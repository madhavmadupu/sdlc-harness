# Security policy

## Reporting a vulnerability

If you discover a security vulnerability, please report it by emailing the
project maintainer rather than opening a public issue.

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

You should receive a response within 48 hours. If the issue is confirmed,
a fix will be released as soon as possible depending on severity.

## Scope

- The `sdlc-harness` npm package
- Knowledge graph data stored in SQLite databases
- Communication with opencode servers
- Personal access tokens used for GitHub Packages authentication

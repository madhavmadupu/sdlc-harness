# Contributing to SDLC Harness

Thank you for considering contributing! This document outlines the process.

## How to contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix
3. **Make your changes** following the project's code style
4. **Write or update tests** as needed
5. **Run tests** — `npm test`
6. **Type-check** — `npm run typecheck`
7. **Submit a pull request**

## Code style

- TypeScript with strict mode
- ESM modules (no CommonJS imports)
- No semicolons
- Descriptive variable names
- Follow existing patterns in the codebase

## Pull request guidelines

- Keep PRs focused on a single concern
- Write a clear title and description
- Reference any related issues
- Ensure CI passes

## Development setup

See [docs/development/setup.md](docs/development/setup.md).

## Reporting issues

Open an issue on GitHub with:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Environment details (Node version, OS, etc.)

## Code of conduct

Please note that this project adheres to a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold its terms.
